
# GitHub.com para arquitectos

## Repositorios y buenas prácticas
- `CODEOWNERS`, plantillas de PR/Issue, *branch protection*.
- GitHub Projects para Roadmaps y Releases.

## Revisión con Copilot en PRs
- Resumen de cambios, riesgos y sugerencias automáticas.

## Paso a paso (mínimo)
1. Crea `.github/PULL_REQUEST_TEMPLATE.md`  
2. Crea `.github/ISSUE_TEMPLATE/bug_report.md` y `feature_request.md`  
3. Configura *branch protection* en `Settings -> Branches`
